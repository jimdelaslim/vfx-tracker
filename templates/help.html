VFX Shot Tracker Help

Getting Started

1
Create a Project
Click "Create New Project" and enter a name and frame rate (FPS). You can change these later in Settings.
2
Import Your EDL
Go to "Import EDL" and upload your CMX3600 EDL file. Shots will be automatically grouped by VFX code.
3
Import Metadata (Optional)
Upload camera metadata CSV files to auto-populate camera, lens, and color information.
4
Assign Plate Numbers & Edit
Assign a plate number to each plate via the dropdown box next to the clip name. You can now select this plate number in the edit section to adjust handles, crank and update timecodes.

Using the Edit Section

The Edit Section (right side) is where you modify individual plates:

1. Select a Plate
Select the plate you wish to edit via choosing the plate number you assigned it in the dropdown box
2. Edit Values
Change start frame, head/tail handles, crank speed, or version
3. Auto-Save
Values save automatically when you click out of a field (on blur)
4. Update Timecodes
Click the green "Update Timecodes" button to recalculate TC Scan In/Out based on your handle changes
Important: When you change handles, the frame numbers update immediately, but the TC Scan In/Out timecodes require clicking "Update Timecodes" to recalculate.

Clip Naming Conventions

Clip names follow a flexible pattern:

PROJECT_SCENE_SHOT_[plateType][plateNum]_v[version]
Examples: WILD_038_0010_bg01_v1  |  WILD_038_0010_src01_v002  |  101_BOR_070_002_bgr001_v1
VFX Code
Everything before the plate type (e.g. WILD_038_0010). Groups all plates for a single shot together.
Plate Type
Any 2-4 letter abbreviation (teal badge). Common types include:
bg
Background
fg
Foreground
src
Source
ref
Reference
pl
Clean Plate
fx
VFX Balls/Charts
bgr
Background (alternate)
fgr
Foreground (alternate)
Plate Number
Any digit count after the plate type: 01, 001, 0001, etc.
Version
Any digit count after _v: v1, v01, v002, etc. (grey badge)
Flexible system: The tracker accepts any 2-4 letter plate type abbreviation - you are not limited to the common types listed above. Use whatever naming convention your project requires.
Editable fields: Clip name, plate type, and version are all editable directly on the dashboard. When you change the plate type or version, the clip name auto-regenerates. When you edit the clip name directly, the plate type badge updates to match.

Tip: Click the status counts in the dashboard header to filter VFX codes by status. The frame range boxes will also match the status color.

Project Settings

Access Settings from the navigation menu to configure:

Project Name
Change the project name. You can leave it empty to show only the logo. Use the "Hide Title on Dashboard" button to toggle title visibility.
Project Logo
Upload a logo image to display in the header alongside or instead of the project title. Both logo and title can be shown together. Use "Hide Logo on Dashboard" to toggle logo visibility.
Frame Rate (FPS)
Set the project's timeline frame rate. All EDL imports will use this FPS for timecode calculations.
Project FPS: This should match your editing timeline's frame rate (23.976, 24, 25, 29.97, 30, etc.), NOT your source footage frame rate.

Crank Speed & Handles

What is Crank Speed?

Crank speed indicates if the footage was shot at a different frame rate than the timeline. This affects both the duration and handles needed.

100%
Normal speed (e.g., 24fps shot at 24fps timeline)
200%
Double speed / Slow motion (e.g., 48fps shot at 24fps timeline)
50%
Half speed / Fast motion (e.g., 12fps shot at 24fps timeline)

How Handles Work with Crank Speed

Handle Input Boxes = SOURCE HANDLES

When you enter handle values, you're specifying how many source frames you need:

Example at 200% Crank (Slow Motion):
• You enter: 16 frames in head handles
• Timeline sees: 8 frames (because 16 source frames ÷ 2 = 8 output frames)
• Vendor receives: 16 source frames to work with

Frame Range Display

1001 [8] 1008 • 1009 [170] 1178 • 1179 [8] 1186
Head Handles • Action • Tail Handles

What This Means:

1001 [8] 1008
Head Handles: 8 frames of OUTPUT handles before the action starts (frames 1001-1008)
1009 [170] 1178
Action Frames: 170 OUTPUT frames of the actual shot (frames 1009-1178)
1179 [8] 1186
Tail Handles: 8 frames of OUTPUT handles after the action ends (frames 1179-1186)
TOTAL: 186
Total frames to deliver to vendor (8 + 170 + 8)
Key Point: These numbers represent OUTPUT frames - what the vendor will deliver back to you after any respeed is applied. This matches your timeline frame numbers.

Starting Frame: 1001

All frame ranges start at frame 1001 by convention. You can change this per-plate in the Edit Section if needed.

Timecode Information

TC Cut In / Out
16:04:01:19 to 16:04:05:00
77 frames - Action only

The timecode range of just the action from the EDL (no handles).

TC Scan In / Out
16:04:01:03 to 16:04:05:16
93 frames - With handles

The full timecode range including handles that the vendor needs to scan.

Edit Section Indicator: When you change handles, the Edit Section background turns yellow to indicate timecodes need updating. Click the green "Update Timecodes" button to recalculate.

Metadata Boxes

The metadata boxes show camera and color information imported from your camera reports:

Lens/Camera
Camera, lens, focal length, aperture, ISO, resolution, tape name, shutter, tilt/roll, distance
Color & LUT
LUT applied, color space, gamma, codec

When no metadata has been imported, the boxes appear grey with "Metadata not imported".

How to import metadata: Go to Import Metadata in the navigation and upload either a CSV or ALE file with your camera reports. Use the dropdown boxes to map your metadata columns to the VFX Tracker's fields. The system will automatically match clips by their cam roll/tape name.

Internal Notes

Each VFX code has an internal notes panel for tracking information that should NOT be exported to vendors.

How to use: Click the "Notes ▶" tab on the right side of any VFX code header. A panel will slide out from the right side of the screen where you can type notes.

Notes auto-save after 1 second of typing and are for internal use only - they will not appear in any PDF exports.

Use cases: Track internal discussions, note revision history, flag issues that vendors shouldn't see, or keep production notes separate from turnover information.

Batch Changes

Update TO#, TO Date, and Vendors across multiple VFX codes simultaneously:

  1. Select VFX Codes: Check the checkbox in the header of 2 or more VFX codes you want to link.
  2. Fields Turn Yellow: Once 2+ codes are selected, the TO#, TO Date, and all 4 Vendor fields will highlight in yellow to show they're linked.
  3. Edit Once, Update All: Change any of these fields in one VFX code and all selected codes will update automatically.
  4. Unlink: Uncheck any checkbox to break the link for that VFX code.
Tip: Use "Select All" to link all visible VFX codes at once. This respects your current search filter, so you can search first, then select all matching results.

Search & Filter Functions

The search bar at the top of the dashboard is powerful and searches across multiple fields:

VFX Code
Search by shot number (e.g., "0010" finds WILD_038_0010)
Clip Name
Search by plate name (e.g., "bg01" finds all background plates)
TO Number
Search by turnover number (e.g., "TO001")
TO Date
Search by turnover date in DD/MM/YY format (e.g., "31/01/26")
Vendor
Search by vendor name across all 4 vendor fields (e.g., "Framestore")

Additional Features:

Select All
Selects all visible VFX codes based on current search/filter
Deselect All
Clears all selections (visible and hidden)
Status Filter
Click the status count boxes (Prep, Ready, Turned Over, etc.) at the top to filter by status
Sort
Use the "Sort by..." dropdown to organize by VFX code, TO#, or TO Date
Pro Tip: Combine search + filter + select all to quickly update groups of shots. For example: search "framestore" → select all → batch update TO date for all Framestore shots.
